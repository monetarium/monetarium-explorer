# Спецификация Monetarium Explorer

Настоящий документ определяет требования к функциональности и интерфейсу обозревателя блоков Monetarium, построенного на базе форка `dcrdata`.

## 1. Технические основы и ограничения

- **Поддержка токенов:** Система должна обеспечивать корректную работу и отображение до **255 типов монет SKA**.
- **Разрядность и точность данных:**
  - **Монеты VAR:** Используют точность 8 знаков до и после запятой.
  - **Монеты SKA:** Имеют повышенную разрядность — **15 знаков в целой части и 18 знаков после запятой**.
  - **Особенности реализации Go:** Стандартные типы данных Go не поддерживают такую точность, поэтому для всех расчетов на бэкенде используется **специализированный модуль**.

## 2. Стандарты интерфейса (UI)

### 2.1. Формат отображения количеств (Главная страница)

Для всех типов монет (**VAR** и **SKA**) на главной странице вводится **единый формат отображения суммы**, аналогичный текущему формату DCR:

- **Правило трех цифр:** Значение суммы отображается с использованием **только трех цифр** (включая десятичную точку, если она необходима).
- **Буквенные сокращения:** Для обозначения порядков величин используются буквы: **K** (тысячи), **M** (миллионы), **B** (миллиарды), **T** (триллионы).
- **Примеры отображения:** `1.25`, `14.8`, `136`, `1.44K`, `15.6K`, `454K`, `1.18M`, `16.5M`, `846M`, `4.89B`, `36.4B`, `789B`, `8.16T`, `34.5T`, `900T`.

### 2.2. Отображение на страницах деталей

- В деталях блоков и транзакций необходимо отображать **все знаки после запятой** (8 для VAR и 18 для SKA) для обеспечения полной прозрачности данных.

### 2.3. Адаптивность и темы

- Интерфейс разрабатывается по принципу **mobile-first**.
- Все новые элементы должны корректно отображаться в **темной теме**.

## 3. Главная страница: Список блоков

Таблица расширяется до **13 колонок**, разделенных на три секции.

### Структура колонок:

1.  **Overview:** Height, Transactions, Voters, Tickets, Revokers, Size, Age.
2.  **VAR:** Transactions, VAR (сумма в формате 3 цифр), Size.
3.  **SKA:** Transactions (общее кол-во), SKA (суммарный объем в формате 3 цифр), Size.

### Интерактивный Аккордеон (Расшифровка SKA):

- **Логика:** В основной строке отображается **сумма** по всем типам SKA.
- **Раскрытие:** При клике на числовые значения в разделе SKA (Transactions, Sum или Size) под основной строкой появляются детали по каждому типу монеты (SKA1, SKA2...).
- **Оформление:** В раскрытых строках **отсутствует номер блока** в колонке Height, чтобы визуально связать их с родительским блоком.

## 4. Виджеты и дополнительные разделы

- **Coin Supply:** Виджет отображает циркулирующий объем VAR и детали по каждой монете SKA: **Issued** (эмитировано), **Withdrawn** (сожжено) и **Circulating** (в обращении).
- **Mempool:** Для идентификации транзакций SKA используются **цветные скобки**.
- **Скрытие данных:** Разделы **Treasury** и **Exchange Rate** скрыты до момента реализации соответствующей логики на бэкенде.

## 5. Техническая реализация

- **Бэкенд:** Go, PostgreSQL 11+, работа через RPC с поддержкой `--txindex`.
- **Фронтенд:** Turbo Streams для динамической подгрузки строк аккордеона и Stimulus для управления состоянием UI.
- **Сборка:** Webpack для компиляции статических ресурсов.
