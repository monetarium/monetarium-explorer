# Спецификация Monetarium Explorer

Настоящий документ определяет требования к функциональности и интерфейсу обозревателя блоков Monetarium, построенного на базе форка `dcrdata`.

## 1. Технические основы и ограничения

- **Поддержка токенов:** Система должна обеспечивать корректную работу и отображение до **255 типов монет SKA**.
- **Разрядность и точность данных:**
  - **Монеты VAR:** Используют точность, аналогичную DCR (8 знаков до и после запятой).
  - **Монеты SKA:** Имеют повышенную разрядность — **15 знаков в целой части и 18 знаков после запятой**.
  - **Особенности реализации Go:** Стандартные типы данных Go не поддерживают такую точность, поэтому для всех расчетов на бэкенде используется **специализированный модуль**, позволяющий обрабатывать числа большой разрядности.
- **Производительность:** Ядро системы должно обеспечивать **быструю начальную загрузку страницы** даже на устройствах с низкой мощностью или в условиях высокой задержки сети.

## 2. Стандарты интерфейса (UI)

- **Отображение сумм SKA:**
  - **На главной странице:** Для экономии места суммы SKA отображаются с точностью до **второго знака после запятой**.
  - **На страницах деталей:** В деталях блоков и транзакций отображаются все **18 знаков** после запятой.
- **Адаптивность:** Интерфейс разрабатывается по принципу **mobile-first**; широкие таблицы должны корректно трансформироваться или прокручиваться на мобильных устройствах.
- **Визуальные темы:** Все новые элементы интерфейса (колонки, аккордеоны) должны поддерживать **темную тему** оформления.

## 3. Главная страница: Список блоков

Таблица «Latest Blocks» расширяется до **13 колонок**, сгруппированных в три смысловых раздела.

### Структура колонок:

1.  **Overview:** Height (Высота), Transactions (Транзакции), Voters, Tickets, Revokers, Size (Размер), Age (Время).
2.  **VAR:** Transactions, VAR (Сумма монет), Size (Размер в блоке).
3.  **SKA:** Transactions (Общее кол-во), SKA (Суммарный объем), Size (Общий размер).

### Интерактивный Аккордеон (Расшифровка SKA):

- **Логика отображения:** В основной строке блока колонки раздела SKA показывают **суммарные значения** по всем типам токенов (SKA1 + SKA2 + ...).
- **Механизм активации:** При клике на **числовые значения** (Transactions, Sum или Size) в разделе SKA строка «раздвигается».
- **Отображение деталей:** Под основной строкой появляются дополнительные строки по количеству типов SKA, присутствующих в данном блоке.
- **Стиль подстрок:** Эти строки должны выделяться визуально (цветом или шрифтом), а в колонке Height у них **отсутствует номер блока**, чтобы подчеркнуть их принадлежность к родителю.

## 4. Виджеты и дополнительные разделы

- **Coin Supply (Эмиссия):** Раздел «Distribution» переименовывается в **«Coin Supply»**.
  - Виджет отображает общее количество циркулирующих монет **VAR** и их процент от целевого показателя.
  - Для каждой монеты **SKA** выводятся параметры: **Issued** (эмитировано), **Withdrawn** (отозвано/сожжено) и **Circulating** (в обращении).
- **Mempool (Мемпул):**
  - Для идентификации разных типов SKA в общем списке транзакций используются **цветные скобки**.
  - Информация о типе и количестве монет выводится непосредственно в строке транзакции для минимизации изменений в верстке.
- **Скрытие неактивных данных:** Временно скрываются разделы **Treasury** (Казначейство) и **Exchange Rate** (Курс обмена), так как они не используются на текущем этапе.

## 5. Техническая реализация

- **Бэкенд:** Написан на **Go**, взаимодействует с нодой через RPC и хранит данные в **PostgreSQL 11+**. Для работы требуется запуск ноды с флагом `--txindex`.
- **Фронтенд:** Используется связка **Turbo** (для обновления DOM без перезагрузки страниц) и **Stimulus** (для управления поведением UI, например, аккордеоном).
- **Динамические обновления:** Раскрытие аккордеона реализуется через **Turbo Streams**, что позволяет серверу на Go «подпушивать» фрагменты HTML-кода напрямую в таблицу по запросу.
- **Сборка:** Компиляция статических ресурсов (CSS/JS) осуществляется через **Webpack**.
