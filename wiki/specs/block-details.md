# Block Page — UI Change Requirements

## 1. Block Card (верхняя левая карточка)

### 1.1. Total Sent

Текущий вид показывает одну строку `Total Sent` в VAR.
Необходимо заменить её на **вертикальный список значений по типам монет**.

**Правила отображения:**

* Значения отображаются списком:

  * `VAR` — всегда первая строка
  * `SKA1`, `SKA2`, … — ниже, только если присутствуют в блоке
* Если SKA-транзакции отсутствуют — отображается только строка `VAR`
* `Mixed` удаляется из UI (не отображается)

**Пример:**

```
Total Sent

VAR   4,973.98183244
SKA1  123.000000000000000001
SKA2  0.000000000000000123
```

---

### 1.2. Счётчик типов транзакций (Regular)

Текущий вид: `Regular: 6`.

Необходимо заменить на **вертикальную разбивку по типам монет**, если присутствует более одного типа.

**Правила отображения:**

  ```
  Regular:  VAR   12,345
            SKA1  8,921
            SKA2  70,002
  ```
* Список выравнивается вертикально
* Значения отображаются с разделителями тысяч
* Используется моноширинный шрифт для чисел (для выравнивания)
* Порядок:

  * VAR всегда первая
  * SKA далее по порядку (SKA1 → SKA2 → …)

**Важно:**

* Votes, Tickets, Revocations не изменяются (всегда VAR) и располагаются на одном уровне с VAR

---

## 2. Block Details (правая верхняя карточка)

### 2.1. Fees (все типы монет)

Текущий вид отображает `Fees` только для VAR в одной строке.

Необходимо заменить на **вертикальный список комиссий по типам монет внутри той же ячейки**.

**Правила отображения:**

* Формат — вертикальный список:

  * `VAR` — первая строка
  * `SKA1`, `SKA2`, … — ниже (если есть)
* Не создавать отдельную таблицу
* Не добавлять новые колонки
* Отображение остаётся внутри текущей карточки Block Details

**Пример:**

```
Fees: VAR   0.01367045
      SKA1  12.000000000000000123
      SKA2  0.500000000000000000
```

**Форматирование:**

* VAR — 8 знаков после запятой
* SKA — 18 знаков после запятой
* SKA значения отображаются как строки (без преобразования в float)

## 5. Таблица Tickets

---

## 6. Таблица Stake Fees

Колонки `Total` и `Fee`:

* Добавить суффикс типа монеты к значениям:

  * `VAR`
  * `SKA{n}`

**Пример:**

```
0.02608455 VAR
0.0067597 SKA1
```

---

## 7. Таблица Transactions

### 7.1. Удалить колонку

* `Mixed`

---

### 7.2. Переименовать колонку

* `Total DCR` → `Total`

---

### 7.3. Формат значений

* `Total`:

  * Добавить суффикс типа монеты (`VAR`, `SKA1`, …)
* `Fee`:

  * Уже содержит суффикс → оставить
* `Fee Rate`:

  * Без изменений (`atoms/B`)

---

### 7.4. Итоговый порядок колонок

```
Transaction ID | Total | Fee | Fee Rate | Size
```

---

### Порядок монет

Во всех местах UI:

1. VAR
2. SKA1
3. SKA2
4. …

---

### Условное отображение

* SKA отображаются **только если присутствуют в блоке**
* Нулевые значения SKA не показываются
