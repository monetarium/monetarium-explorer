## Block Page — UI Change Requirements

### 1. Block Card (верхняя левая карточка)

**1.1. Total Sent**
Текущий вид показывает одну строку `Total Sent` в VAR. Необходимо добавить отдельную строку `Total Sent` для каждого типа SKA-монеты, присутствующей в блоке. Строки SKA отображаются ниже строки VAR. Если блок не содержит SKA-транзакций, дополнительные строки не отображаются.

**1.2. Счётчик типов транзакций**
Текущий вид: `Regular: 6`. Необходимо добавить разбивку по типам монет в скобках: `Regular: 6 (4 VAR, 2 SKA1)`. Разбивка отображается только если в блоке присутствуют Regular-транзакции более чем одного типа монет. Если все Regular-транзакции одного типа — отображается только общее число без скобок. Votes, Tickets, Revocations не изменяются — они всегда в VAR.

---

### 2. Block Details (правая верхняя карточка)

**2.1. Fees (все типы монет)**

Текущий вид отображает значение `Fees` только для VAR внутри карточки Block Details. Необходимо удалить отображение `Fees` из этой карточки и заменить его на отдельную таблицу, содержащую комиссии по всем типам монет.

Новая таблица располагается непосредственно под двумя верхними карточками, над таблицей Block Reward.

**Заголовок таблицы:** `Fees`

**Структура таблицы:**

- Каждая строка соответствует одному типу монеты:
  - `VAR`
  - `SKA1`, `SKA2`, … (только при наличии в блоке)

- Колонки:
  - `Coin` — тип монеты
  - `Fees` — суммарные комиссии в данной монете

---

**Правила отображения:**

- Строка `VAR` всегда отображается первой.
- Строки SKA отображаются ниже, отсортированные по индексу (SKA1 → SKA2 → …).
- Если в блоке отсутствуют SKA-транзакции — таблица содержит только одну строку (`VAR`).
- Значения:
  - VAR — 8 знаков после запятой
  - SKA — 18 знаков после запятой

**Пример:**

```
Fees

Coin    Fees
VAR     0.01367045
SKA1    12.000000000000000123
SKA2    0.500000000000000000
```

### 6. Таблица Stake Fees

Колонки `Total` и `Fee` не имеют явного указания типа монеты в заголовке. Необходимо добавлять суффикс с типом монеты к каждому значению в ячейках `Total` и `Fee` по аналогии с тем, как уже реализован суффикс `VAR` в колонке Fee. Для SKA-транзакций суффикс принимает вид `SKA{n}`.

---

### 7. Таблица Transactions

**7.1. Убрать колонку `Mixed`** — функциональность не используется.

**7.2. Переименовать колонку `Total DCR`** → `Total`.

- `Total` — сумма транзакции. Необходимо добавить суффикс типа монеты к значению (по аналогии с текущим суффиксом `VAR` в колонке Fee).

**7.3. Колонки `Fee` и `Fee Rate`** — значения отображаются в монете транзакции. Суффикс типа монеты добавляется к значениям в колонке `Fee`. Колонка `Fee Rate` сохраняет единицу измерения `atoms/B` для всех типов монет без изменений.

**Итоговый порядок колонок:** `Transaction ID | Total | Fee | Fee Rate | Size`
