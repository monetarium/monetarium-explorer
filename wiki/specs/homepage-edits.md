# Правки главной страницы — Homepage

## 1. Общее форматирование чисел для монет (везде на главной)

- Единый принцип: **до сотых включительно — крупный шрифт**; остальные разряды — мелкий
- Сотые **всегда присутствуют**: если значение целое, показывать `.00` крупным шрифтом
- Хвостовые нули (после сотых) — серым цветом, везде кроме Coin Supply; для Coin Supply - нули после сотых не отображать
- Убрать разнобой: сейчас где-то только целые, где-то десятые мелким — привести к единому виду

## 2. Секция Voting

Все числовые значения монет в секции отображать с форматированием из пункта 1

### Current Ticket Price

Остается без изменений

### Ticket Pool Size

Остается без изменений

### Next Ticket Price

Остается без изменений

### Next Ticket Price Change

Остается без изменений

### Vote VAR Reward

- Заголовок: добавить **(most recent)** → итоговый вид: **Vote VAR Reward (most recent)**
- Строка `VAR/VOTE` - Остается без изменений (простое отображение значения с форматированием)
- Добавить расшифровку под значением:
  - строка **субсидии** (доля от общей субсидии блока, делённая на количество проголосовавших билетов)
  - строка **fee** (доля комиссии, приходящаяся на один билет)
  - строка **ROI** (годовая рентабельность)

  ```text
  Subsidy: 3.20000000
  Fee: 0.01256400
  ROI: 12.02% per year
  ```

### Total Staked VAR

Остается без изменений

### Vote SKA Reward

- Заголовок: добавить **Fee** и **(most recent)** → итоговый вид: **Vote SKA Fee Reward (most recent)**
- Числовое значение `SKA{n}/VOTE` отобразить как гиперссылку на страницу соответствующего блока
- Под значением добавить доходность `SKA{n}/VAR per year` (Мелким шрифтом светло-серого цвета). Значение `SKA{n}/VAR per 30 days` убрать

## 3. Секция Mining

Все числовые значения монет в секции отображать с форматированием из пункта 1

### Difficulty

Убедится что значение отображается по такому правилу: Difficulty отображаем все значимые числа. Предположительно, после запятой их может быть до 8 разрядов. Не отображаем хвостовые нули. Но первые два разряда после запятой отображаем всегда.

### Hashrate

Остается без изменений

### PoW VAR Reward

- Изменить заголовок на **PoW VAR Reward (most recent)**
- Добавить расшифровку под значением:
  - строка **субсидии**
  - строка **fee**

  ```text
  Subsidy: 3.20000000
  Fee: 0.00364000
  ```

### Next Block VAR Reward Reduction

- Изменить заголовок → итоговый вид: **Next Block Subsidy Reduction**

### PoW SKA Reward

- Заголовок: добавить **Fee** и **(most recent)** → итоговый вид: **PoW SKA Fee Reward (most recent)**
- Числовое значение `SKA{n}` отобразить как гиперссылку на страницу соответствующего блока

## 4. Секция Coin Supply

Все числовые значения монет в секции отображать с форматированием из пункта 1, учитывая особенности форматирования для Coin Supply.

- Убрать подзаголовки **VAR Coin Supply**, **SKA Coin Supply**
- Добавить значение доли эмитированных монет от максимальной эмиссии для VAR в таком формате:

  ```text
  3.2% of ~54M
  ```

- Иконка: визуально чуть увеличить (примерно на 10%), чтобы она воспринималась того же размера, что иконки Mining и Staking секций — **только размер иконки, шрифт заголовка не трогать**
