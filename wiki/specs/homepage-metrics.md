# Спецификация Главной страницы (Homepage)

Этот документ определяет требования к интерфейсу и функциональности главной страницы Monetarium Explorer.

---

## 1. Общие правила отображения

### 1.1. Правило трех цифр

Для VAR и SKAN на главной странице суммы отображаются с использованием **только трех значащих цифр** и буквенных сокращений (K, M, B, T).

### 1.2. Coins Supply — исключение из правила трёх цифр

Формат отображения значений соответствует правилам п. **4.3. Раздел Coin Supply**

## 2. Таблица Latest Blocks

Таблица предназначена для отображения списка записей с агрегированными значениями и связанными транзакциями с возможностью раскрытия деталей на уровне строки.

---

### 2.1. Область отображаемых данных

#### 2.1.1. Основной уровень (строка)

Каждая строка должна содержать следующие поля:

---

##### 2.1.1.1. Height

- **Сокращённый заголовок:** Height
- **Подсказка:** block height
- **Описание:** Порядковый номер блока

---

##### 2.1.1.2. Transactions

- **Сокращённый заголовок:** Txn
- **Подсказка:** number of transactions
- **Описание:** Количество транзакций за вычетом транзакций голосования (Vote), покупки билетов (Tickets) и возврата билетов (Revocations) — то есть все типы транзакций, не отражённые в остальных столбцах таблицы. Включает: обычные перечисления VAR, субсидию майнеру (VAR), субсидию голосующим (VAR), перечисления SKA, а также эмиссию монет (редкое событие).

---

##### 2.1.1.3. VAR

- **Сокращённый заголовок:** VAR
- **Подсказка:** total VAR amount
- **Описание:** Общая сумма перечисленных монет VAR

---

##### 2.1.1.4. SKA

- **Сокращённый заголовок:** SKA
- **Подсказка:** total SKA amount
- **Описание:** Общая сумма перечисленных монет SKA
  Если сумма равна **0**, отображается **0**, а не прочерк.

---

##### 2.1.1.5. Size

- **Сокращённый заголовок:** Size
- **Подсказка:** total block size
- **Описание:** Общий (суммарный) размер

---

##### 2.1.1.6. Votes

- **Сокращённый заголовок:** Vote
- **Подсказка:** tickets voted
- **Описание:** Количество билетов, проголосовавших за включение данного блока в блокчейн

---

##### 2.1.1.7. Tickets

- **Сокращённый заголовок:** Tkt
- **Подсказка:** tickets purchased
- **Описание:** Количество купленных билетов для голосования

---

##### 2.1.1.8. Revokes

- **Сокращённый заголовок:** Rev
- **Подсказка:** tickets revoked
- **Описание:** Количество возвращённых (аннулированных) билетов по причине неактивности кошелька данного билета во время голосования или истечения срока голосования для данного билета

---

##### 2.1.1.9. Age

- **Сокращённый заголовок:** Age
- **Подсказка:** block age
- **Описание:** Время, прошедшее от момента включения этого блока в блокчейн до настоящего момента.

---

#### 2.1.2. Заголовки столбцов — отображение полных названий

В заголовке таблицы сокращения Tkt, Rev следует заменить на полные слова (Tickets, Revokes) при наличии достаточного места. На мобильных устройствах — использовать сокращённые формы. Всплывающие подсказки (tooltip) остаются без изменений.

---

### 2.2. Уровень транзакций (expanded rows)

- Expanded rows используют **те же колонки**, что и основной уровень.
- Отдельные заголовки для них не предусмотрены.

---

#### 2.2.1. VAR строка

Значения должны интерпретироваться следующим образом:

- **Transactions (Txn):**
  Описание: Количество транзакций простого перечисления монет VAR

- **VAR:**
  Описание: Общая сумма перечисленных монет VAR

- **Size:**
  Описание: Суммарный размер транзакций VAR

---

#### 2.2.2. SKAN строки

Для каждой строки SKAN (SKA1, SKA2, …) значения интерпретируются:

- **Transactions (Txn):**
  Описание: Количество транзакций простого перечисления монет SKAN

- **SKA:**
  Описание: Общая сумма перечисленных монет SKAN
  Если значение равно **0**, отображается **0**, а не прочерк.

- **Size:**
  Описание: Суммарный размер транзакций SKAN

---

#### 2.2.3. Иерархия данных

Данные должны быть организованы в два уровня:

1. Основной уровень (строка)
2. Expanded rows:
   - одна строка VAR
   - множество строк SKAN
     - если монета **SKAN эмитирована**, её строка **всегда отображается в аккордеоне**, даже если:
       - в блоке отсутствуют транзакции этой монеты
       - значения равны 0

Дополнительные уровни вложенности не допускаются.

---

#### 2.2.4. Поведение раскрытия (expand/collapse)

##### 2.2.4.1. Раскрытие строки

- Пользователь должен иметь возможность раскрыть строку
- При раскрытии отображаются:
  - VAR
  - **все эмитированные SKAN**

##### 2.2.4.2. Состояние по умолчанию

- Строки должны быть свернуты

---

### 2.3. Подсчёт агрегированных значений

На основном уровне:

- Transactions = все транзакции за вычетом Vote, Tickets, Revokes
- VAR = сумма VAR транзакций
- SKA = сумма всех SKAN

---

### 2.4. Отсутствующие данные

Если значение отсутствует — должно отображаться явное обозначение отсутствия.

- Для числовых значений:
  - используется **0**, если значение равно нулю
- Прочерк используется **только если значение неприменимо для данного типа строки** например, количество SKAN-транзакций для монеты VAR

---

### 2.5. Уникальность и идентификация

- Каждая строка имеет уникальный Height
- Каждая запись связана с одной строкой
- Каждый SKAN идентифицируется уникальным номером N

---

### 2.6. Сортировка

- Основные строки сортируются по Height
- SKAN строки сортируются по номеру n

---

### 2.7. Масштабируемость

Система должна поддерживать:

- большое количество строк
- до 255 SKAN на строку

---

### 2.8. Навигация

- Height должен поддерживать переход к деталям блока

---

### 2.9. Ограничения

- Отсутствует агрегированная строка SKA в expanded rows
- Нет вложенности глубже одного уровня
- Все SKAN отображаются при раскрытии

---

### 2.10. Расширяемость

Система должна позволять:

- добавление новых типов транзакций
- расширение набора метрик

---

## 3. Раздел Mempool

Раздел предназначен для мониторинга транзакций в очереди (mempool) и визуализации прогнозируемого заполнения следующего блока.

### 3.1. Изменения в таблице транзакций

Необходимо заменить одиночную колонку VAR на две новые колонки:

- Coin — тип монеты, участвующей в транзакции (VAR, SKA1, SKA2 и т. д.).
- Amount — числовое значение для этой монеты.

Колонка Hash будет сокращена (truncate) еще сильнее, чтобы освободить место для новой колонки — это допустимый компромисс.

### 3.2. Удаляемые элементы

Из раздела Mempool удаляется стандартный анимационный элемент dcrdata, располагавшийся непосредственно под заголовком раздела.

### 3.3. Индикатор распределения места в блоке (новый элемент)

Вместо удалённого элемента (см. §3.2) вводится **набор горизонтальных полос-индикаторов** — одна, отображающая заполненность всего следующего блока (Total) и по одной на каждую монету, реально присутствующую в блокчейне. Каждая полоса подписана именем монеты (VAR, SKA1, SKA2 и т.д.) и отображает степень заполнения гарантированного для неё места в блоке.

#### 3.3.1. Гарантированные лимиты блока

- **VAR:** 10% блока.
- **SKA:** оставшиеся 90% делятся **поровну** между всеми эмитированными типами SKA.
  - 1 тип SKA → 90%
  - 2 типа SKA → по 45% каждой
  - 3 типа SKA → по 30% каждой
  - и т.д.

#### 3.3.2. Что показывает индикатор

Уровень заполнения каждого индикатора показывает, какой **процент гарантированного места** данной монеты занят её транзакциями из мемпула.

#### 3.3.3. Цветовая логика

| Цвет        | Условие                          | Результат            |
| :---------- | :------------------------------- | :------------------- |
| **Зелёный** | В рамках гарантированной доли    | Попадают в блок      |
| **Жёлтый**  | Сверх доли, есть свободное место | Попадают в блок      |
| **Красный** | Сверх доли, места недостаточно   | Часть **не попадёт** |
| **Серый**   | Свободное место в рамках доли    | —                    |

#### 3.3.4. Поведение при смене цвета

При заполнении индикатора до границы гарантированного лимита цвет меняется в соответствии с логикой §3.3.3.

#### 3.3.5. Масштабируемость и видимость

- Индикатор для каждой монеты **добавляется автоматически** при её появлении в блокчейне.
- Индикатор **отображается постоянно** (даже если транзакций данной монеты в мемпуле нет — показывается 0% без цветного заполнения). Индикатор не скрывается и не исчезает после появления.
- Количество одновременно отображаемых индикаторов ограничено возможностями текущего дизайна страницы. В ближайшие годы ожидается эмиссия лишь нескольких монет SKA, поэтому существующий дизайн рассчитан на это количество. При необходимости в будущем дизайн модуля может быть пересмотрен.
- Размеры всех индикаторов одинаковы.
- Размер шрифта для всех индикаторов одинаков (Total, VAR, SKAN, проценты).
- Для индикатора Total используется более темный оттенок серого, чем для индикаторов VAR и SKAN.

## 4. Дополнительные блоки главной страницы

### 4.1. Раздел Voting (Голосование)

Раздел в основном остаётся без изменений. Производятся следующие замены и дополнения:

#### 4.1.1. Vote VAR Reward (замена подраздела Vote Reward)

Подраздел `Vote Reward` переименовывается в `Vote VAR Reward`. Отображает три строки:

- `0.00000000 VAR/Vote` — сумма в монетах VAR, которую получил один конкретный билет, проголосовавший за включение последнего блока в блокчейн.
- `0.00% per 30 days` — усреднённая доходность одной монеты VAR за последние 30 дней.
- `0.00% per year` — усреднённая доходность одной монеты VAR за последние 365 дней.

**Формула расчёта:** реализована в коде dcrdata. Разработчик должен предоставить описание формул для всех трёх строк (особенно за год — там используется симуляция). После уточнения формул возможна корректировка.

#### 4.1.2. Vote SKA Reward (новый подраздел)

Добавляется подраздел `Vote SKA Reward`. Для каждого эмитированного типа SKA отображается блок из трёх строк:

- `SKAN/Vote` — конкретное количество вознаграждений SKAN, распределенных на один билет, участвовавший в голосовании в последнем блоке.
- `SKAN/VAR per 30 days` — усреднённое суммарное SKAN-вознаграждение за одну монету VAR за последние 30 дней.
- `SKAN/VAR per year` — усреднённое суммарное SKAN-вознаграждение за одну монету VAR за последние 365 дней.

Количество таких блоков равно количеству эмитированных типов SKA.

**Поведение при отсутствии SKA в последнем блоке:** отображается значение из последнего блока, в котором были SKA-транзакции. При старте сервера система автоматически находит последнюю транзакцию с выводом SKA за последние 30 дней для инициализации значений.
Строки за 30 дней и год считаются накопительно и отображаются независимо от наличия SKA в текущем блоке.

#### 4.1.3. Расположение в разделе Voting

- `Next Ticket Price` и `Ticket Pool Size` меняются местами (Ticket Pool Size — справа/сверху, Next Ticket Price — слева/второй ряд).
- `Total Staked VAR` перемещается вправо — непосредственно под зелёный индикатор `Next Ticket Price Change`, справа/третий ряд.
- Блоки SKA-вознаграждений располагаются снизу по мере их появления.

---

### 4.2. Раздел Mining (Майнинг)

Раздел в основном остаётся без изменений. Производятся следующие замены и дополнения:

#### 4.2.1. PoW VAR Reward (замена подраздела PoW Reward)

Подраздел `PoW Reward` переименовывается в `PoW VAR Reward`. Отображает одну строку:

- `VAR per last block` — размер VAR-вознаграждения за создание последнего блока.

Заголовок зелёного индикатора сложности меняется на `Next Block VAR Reward Reduction`.

#### 4.2.2. PoW SKA Reward (новый подраздел)

Добавляется новый подраздел `PoW SKA Reward`. Для каждого эмитированного типа SKA — одна строка:

- `SKA1 per last block` — размер SKA1-вознаграждения за создание последнего блока.

Количество строк равно количеству эмитированных типов SKA.

**Поведение при отсутствии SKA в последнем блоке:** отображается значение из последнего блока, в котором были SKA-транзакции. При старте сервера система автоматически находит последнюю транзакцию с выводом SKA за последние 30 дней для инициализации значений.

#### 4.2.3. Расположение в разделе Mining

- Зелёный индикатор сложности `Next Block VAR Reward Reduction` и `PoW SKA Reward` меняются местами: индикатор `Next Block VAR Reward Reduction` — справа/второй ряд, `PoW SKA Reward` — снизу, с возможностью добавления строк при эмиссии новых SKA.

---

### 4.3. Раздел Coin Supply (замена раздела Distribution)

Раздел `Distribution` заменяется разделом `Coin Supply`.
Количество циркулирующих монет VAR и SKA отображается **без округления**, с выводом **всех знаков дробной части до последнего ненулевого знака** (без хвостовых нулей).

Примеры:

- 900,000,000,000,000 SKA1
- 900,000,000,000,000.568796 SKA1
- 900000000000000.560000 → отображается как 900,000,000,000,000.56

#### 4.3.1. VAR Coin Supply

Отображает текущее количество эмитированных монет VAR относительно целевого.

**Изменение:** убрать строку с `target` (вида `2.75 VAR`) — в Monetarium такого параметра нет.

#### 4.3.2. SKA Coins Supply

- Формат монет: **SKAN (SKA1, SKA2, …)**
- Названия монет являются **кликабельными ссылками на графики supply** (например: `/charts?chart=coin-supply`)

Для каждой эмитированной монеты SKA отображается блок:

- **Крупными цифрами** — количество монет в обращении (`In Circulation`), с подписью типа монеты справа (например: `900,000,000,000,000.568796 SKA1`). Строка `In Circulation` как отдельная подпись не выводится — она подразумевается крупным числом.
- Мелким шрифтом — две дополнительные строки:
  - `Issued: <число>` (или `Total Issued`) — всего выпущено
  - `Burned: <число>` (или `Total Burned`) — всего сожжено/выведено из обращения

Количество таких блоков равно количеству эмитированных типов SKA.
