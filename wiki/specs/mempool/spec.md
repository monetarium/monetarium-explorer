# ✅ Мемпул — поддержка SKA монет (requirements)

## Общий принцип

Страница мемпула должна отображать актуальные данные по транзакциям и агрегированным значениям для монет:

- **VAR**
- **SKA (SKA1, SKA2, …)**

Отображаемые данные должны соответствовать **текущему состоянию мемпула**.

---

## Маппинг типов транзакций и монет

- **Regular** — VAR и SKA
- **Ticket, Vote, Revocation** — только VAR

---

## Карточка "Current Mempool" (левая верхняя)

Поле **Total Sent** должно отображать суммы по монетам.

### Правила

- Отображаются:
  - VAR (всегда)
  - SKA — если присутствуют в мемпуле

- Порядок:
  1. VAR
  2. SKA1 … SKA{n} (по возрастанию номера)

- Если для монеты нет транзакций:
  - она не отображается

---

### Точность

- VAR → до 8 знаков после запятой
- SKA → до 18 знаков после запятой

---

### Обновления

- Данные обновляются в реальном времени
- Интерфейс должен отражать актуальное состояние мемпула без перезагрузки страницы

---

## Карточка "Transactions" (правая верхняя)

Карточка отображает агрегированные данные по типам транзакций.

---

### Структура

Карточка должна содержать секции по монетам:

#### VAR:

- Regular
- Tickets
- Votes
- Revocations

#### SKA:

- только Regular

---

### Правила отображения

- Отображаются:
  - VAR (всегда)
  - SKA — если присутствуют в мемпуле

- Для каждого типа транзакций:
  - отображается количество и сумма

- Если для типа транзакций нет данных:
  - отображается `0`

---

### Обновления

- Данные обновляются в реальном времени
- При изменении состава монет в мемпуле:
  - соответствующие секции должны добавляться или удаляться

---

## Таблицы транзакций

### Общие правила

Таблицы отображают **только фактические транзакции**, находящиеся в мемпуле.

- Нули для отсутствующих данных не отображаются
- Если таблица пуста:
  - отображается **empty state**

---

### Empty state

Если в таблице нет транзакций, отображается сообщение:

- Regular: `No transactions in mempool.`
- Tickets: `No tickets in mempool.`
- Votes: `No votes in mempool.`
- Revocations: `No revocations in mempool.`

---

### Общие изменения

- Колонка:
  - `Total DCR` → `Total VAR`

- Таблица **Treasury Spends**:
  - должна быть удалена

---

## Таблица Regular

### Колонки

```
Transaction ID | Coin | Amount | Size | Fee Rate | Time in Mempool
```

---

### Правила

- Каждая транзакция должна содержать:
  - тип монеты (`VAR`, `SKA1`, `SKA2`, …)

- Отображаются только реальные транзакции

---

### Формат суммы

- VAR → до 8 знаков после запятой
- SKA → до 18 знаков после запятой

---

## Обновления в реальном времени

- Все компоненты страницы должны обновляться без перезагрузки:
  - карточки
  - таблицы

- Интерфейс должен всегда отражать текущее состояние мемпула

---

## ⚠️ Ключевая инварианта

- Интерфейс отображает **фактическое состояние мемпула**
- Данные в карточках и таблицах должны быть **консистентны между собой**
