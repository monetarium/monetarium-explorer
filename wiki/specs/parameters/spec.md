# Parameters — приведение страницы `/parameters` к модели Monetarium (требования)

## 0. Контекст

Страница `/parameters` показывает **параметры сети Monetarium**: правила добычи (PoW) и стейкинга (PoS), расписание субсидии, экономику билетов, сетевые константы и адресные префиксы. Раздел унаследован от Decred и в текущем виде местами всё ещё показывает данные и подписи Decred.

Страница почти полностью **справочная и статичная**: практически все значения берутся из конфигурации сети, зафиксированной при запуске обозревателя, и не меняются между блоками (динамически обновляется фактически только максимальный размер блока). Это нормальное поведение, оно сохраняется.

Инварианты Monetarium, на которые опирается документ (см. [core/product.md](../../core/product.md), [core/pages.md](../../core/pages.md)):

- В сети несколько монет: **VAR** и до 255 типов **SKA{n}** (SKA1, SKA2, …).
- Параметры субсидии и стейкинга — величины **только для VAR**; они не «разбиваются по монетам» (см. §8).
- **Treasury (Project Fund) на Monetarium отсутствует** (доля казны = 0; страница `/treasury` отключена).
- Голосование за изменения правил on-chain (агенды) на Monetarium **доступно** — страница `/agendas` включена (PR #395).
- Определения SKA-монет — часть конфигурации сети и должны быть показаны отдельной секцией (см. §9).
- **Точность сумм:** VAR — 8 знаков после запятой; SKA — 18 знаков. SKA-суммы крупнее того, что вмещает обычная дробная арифметика, поэтому показываются полностью, без округления и без «экспоненциальной» записи. На этой странице это правило задействовано в секции SKA-монет (§9).

### 0.1. Происхождение требований и принятые решения

Документ составлен по анализу страницы и сверке с конфигурацией сети `monetarium-node`. Спорные продуктовые вопросы решены так:

| Вопрос                                                  | Решение (принято)                                                                                  |
| ------------------------------------------------------- | -------------------------------------------------------------------------------------------------- |
| Секция «Treasury parameters» + строка доли казны (`BlockTaxProportion`) | **Удалить полностью** — на Monetarium нет казны, доля казны = 0, страница `/treasury` отключена (§4, §5) |
| Секция «Rule change parameters»                          | **Оставить как справочную** — это валидные параметры сети, не привязанные к отключённым агендам (§7) |
| Доли награды PoW / PoS                                    | **Показывать фактические 50 / 50** — консенсус-сплит Monetarium (§5)                                |
| Определения SKA-монет                                    | **Добавить новую секцию** «SKA coin parameters» (§9)                                                |

---

## 1. Структура страницы (общий вид)

Страница — это заголовок и набор таблиц вида «Параметр / Значение / Описание». После правок секции сверху вниз:

1. Заголовок страницы и ссылка на источник параметров (см. §2).
2. **Chain parameters** — сетевые константы (см. §3).
3. **Subsidy parameters** — субсидия (только VAR), без строки доли казны (см. §5).
4. **Stake parameters** — экономика билетов (только VAR) (см. §6).
5. ~~**Treasury parameters**~~ — **удалена** (см. §4).
6. **Rule change parameters** — сохраняется как справочная (см. §7).
7. **Address parameters** — корректные адресные префиксы Monetarium (см. §10).
8. **SKA coin parameters** — **новая** секция (см. §9).

---

## 2. Заголовок страницы и ссылка на источник параметров

**Текущее поведение:**
Заголовок уже корректен — «Parameters for Monetarium …». Рядом стоит ссылка «from chaincfg/params.go» (исходник параметров сети), но она ведёт на репозиторий **Decred**, а не Monetarium.

**Требуемое поведение:**

- Ссылка на источник параметров должна вести на проект **Monetarium**, а не на Decred.
- Сам заголовок страницы менять не нужно — он уже корректен («Parameters for Monetarium …»). Меняется только цель ссылки рядом с ним.

---

## 3. Chain parameters — без смысловых изменений

Строки этой секции (генезис-блок, премайн-блок, лимиты сложности, максимальный размер блока, целевое время блока и т. д.) сохраняются. В частности:

- Максимальный размер блока показывается из актуальных данных сети, а между блоками — из конфигурации сети (это штатное поведение, сохраняется).
- Ссылки на блоки (генезис, премайн) сохраняются.
- Подписи `DCR` на странице нет уже сейчас (суммы подписаны `VAR`); никакой `DCR` не вводить.

---

## 4. Treasury parameters — секция удалена

**Текущее поведение:**
Секция «Treasury parameters» показывает параметры казны (интервалы и кворумы голосований за расходование казны и т. п.).

**Требуемое поведение: секцию удалить полностью.**

**Причина.** На Monetarium казны нет: доля казны в конфигурации сети равна нулю с явной пометкой «Monetarium has no treasury»; страницы `/treasury` и `/treasurytable` отключены; казна нигде в интерфейсе не показывается (см. [core/pages.md](../../core/pages.md)). Значения параметров казны на странице бессмысленны и вводят в заблуждение. Возвращать казну на страницу запрещено.

---

## 5. Subsidy parameters — только VAR, без доли казны, фактические 50 / 50

Секция остаётся **только для VAR** (это VAR-величины сети, см. §8). Стартовая субсидия показывается в **VAR**. Изменения:

1. **Удалить строку доли казны (`BlockTaxProportion`).** На Monetarium она всегда равна 0 % (нет казны и developer-организации), строка неинформативна. Согласовано с §4.

2. **Доли награды PoW / PoS — показывать фактические 50 % / 50 %.**
   - **Текущее поведение:** страница показывает «60 % / 30 %» — это устаревшие исходные доли, унаследованные от Decred.
   - **Проблема:** фактический PoW/PoS-сплит Monetarium — фиксированные **50 / 50**. Это консенсус-правило Monetarium (`SSVMonetarium`, 50 % PoW / 50 % PoS, отдельно по каждой монете), авторитетно зафиксированное в [docs/pow-pos-reward-classification-problem.md](../../../docs/pow-pos-reward-classification-problem.md); тот же результат заложен в актуальной конфигурации сети и используется во всём остальном обозревателе (ср. [REWARDS_LOGIC.md](../../../REWARDS_LOGIC.md)). Текущие «60 % / 30 %» противоречат консенсусу и остальному интерфейсу.
   - **Требуемое поведение:** строки должны показывать фактический сплит **50 % / 50 %**.
   - **Важное разграничение:** правило `SSVMonetarium` определяет, как делятся **комиссии** между майнером и стейкером. Строки же на этой странице — про долю **субсидии блока**. На Monetarium обе величины совпадают (50 / 50), но это два разных параметра; на странице показывается именно доля субсидии. Сам пересчёт наград этой страницей не выполняется — она только отображает долю субсидии.
   - **Источник данных (для разработчиков):** отображаемые доли берутся из фактических (актуальных) пропорций субсидии в конфигурации сети, а не из устаревших исходных полей, дающих «60 % / 30 %».
   - При необходимости уточнить описание строк: на Monetarium сплит PoW/PoS фиксирован 50 / 50.

Остальные строки секции (множитель/делитель субсидии, интервал её снижения) — без изменений.

---

## 6. Stake parameters — без смысловых изменений, только VAR

Стейкинг на Monetarium ведётся **только в VAR**. Секция сохраняется как есть; минимальная цена билета показывается в **VAR**. Ссылки на блоки (высота включения стейкинга, высота начала обязательного голосования) сохраняются. Размеры пула билетов, лимиты на блок и прочие stake-параметры — без изменений (это параметры стейкинга, к мультимонетной модели отношения не имеют).

---

## 7. Rule change parameters — сохраняется как справочная

**Решение:** секция «Rule change parameters» (пороги и кворумы активации изменений правил, требования по версиям блоков) **сохраняется как есть**.

**Причина.** В отличие от казны, это **действующие параметры сети**: они описывают механику смены правил цепи и остаются корректным справочным содержимым. Более того, страница `/agendas` теперь включена (PR #395) и опирается на эти пороги, так что они тем более актуальны. Удалять их не нужно.

---

## 8. Субсидия и стейкинг не «разбиваются по монетам»

Строки субсидии и стейкинга (стартовая субсидия, доли PoW/PoS, минимальная цена билета и т. п.) — это величины сети **только для VAR**, а не наборы значений «по каждой монете». SKA-монеты эти величины не параметризуют.

**Запрещено** превращать эти строки в таблицы «по монетам» («разбивка субсидии по SKA»). Единственное мультимонетное содержимое страницы — **отдельная** секция определений SKA-монет (§9): она описывает конфигурацию монет, а не «делит субсидию по монетам».

---

## 9. SKA coin parameters — новая секция

**Требуемое поведение:** добавить новую таблицу «SKA coin parameters» — перечень SKA-монет, сконфигурированных в сети, по одной строке/блоку на монету, в порядке номера типа монеты.

Источник данных — определения SKA-монет из конфигурации сети. На каждую монету показать:

| Поле                    | Содержание                                                                                                  |
| ----------------------- | ----------------------------------------------------------------------------------------------------------- |
| Метка монеты            | в формате **`SKA{n}`** (например `SKA1`), **без дефиса** — единый формат Monetarium (см. [core/product.md](../../core/product.md)). Не использовать узловой символ вида `SKA-1`. |
| Имя (`Name`)            | как есть (например `Skarb-1`)                                                                               |
| Активность (`Active`)   | активна / не активна; визуально различать (с учётом списка изначально активных монет)                       |
| Описание                | как есть                                                                                                    |
| Максимальная эмиссия    | полная сумма (18 знаков SKA), показывается целиком, без округления и без экспоненциальной записи             |
| Атомов в монете         | масштаб монеты (например 1e18)                                                                              |
| Минимальная комиссия    | сумма SKA, по тому же правилу точности                                                                       |
| Множитель макс. комиссии | целое                                                                                                      |
| Высота начала эмиссии   | целое; ссылка на блок этой высоты                                                                            |
| Окно эмиссии            | целое (в блоках)                                                                                            |
| Адреса эмиссии          | каждый адрес — ссылка на страницу адреса                                                                     |

**Не показывать в v1:** ключ эмиссии (технический публичный ключ, в конфигурации помечен как dev-заглушка — пользователю бесполезен, только зашумляет). Суммы эмиссии — показывать опционально, если помещаются; иначе достаточно максимальной эмиссии.

**Точность SKA (важно).** Максимальная эмиссия, минимальная комиссия, масштаб монеты, суммы эмиссии — это 18-знаковые SKA-величины. Они крупнее обычной дробной арифметики, поэтому показываются как точные значения целиком (без округления, без «экспоненциальной» записи). Метка монеты — строго в формате `SKA{n}` без дефиса.

## 10. Address parameters — корректные префиксы Monetarium

**Текущее поведение (ошибка):**
Таблица адресных префиксов показывает **префиксы Decred** (`Dk / Ds / …`), тогда как у Monetarium-mainnet адресный префикс — `M`, а обычные адреса начинаются с `Ms`. Это тихая ошибка: страница отдаётся без сбоя, но значения неверные. Дополнительно, для сетей, не распознанных текущей логикой (например dev-сеть), таблица адресов может оказаться **пустой без какого-либо сообщения**.

**Требуемое поведение:**

- Адресные префиксы должны соответствовать **фактической сети Monetarium** (mainnet → `M` / `Ms…`; аналогично для testnet, simnet и любой используемой dev-сети). Предпочтительно брать их из самой конфигурации сети, а не из захардкоженной таблицы Decred — это разом устраняет и неверные значения, и проблему «пустой таблицы» для нераспознанной сети.
- Строка `NetworkAddressPrefix` уже берётся напрямую из конфигурации сети и корректна — её не трогать.

---

## 11. Поведение страницы (сохранить)

- Страница справочная и статичная: значения отражают конфигурацию сети на момент запуска обозревателя. Изменение конфигурации сети отражается здесь только после перезапуска обозревателя — это ожидаемое поведение, не ошибка.
- Если базовые данные страницы недоступны (например, недоступна БД), показывается штатная страница ошибки — это поведение сохраняется; новые секции не должны его ломать.
- Кеширование страницы по блокам сохраняется; «не меняющееся» содержимое между блоками — ожидаемо (двигается только максимальный размер блока).

## 13. Вне области действия

- Страница `/parameters` остаётся включённой; пункт меню сохраняется.
- Живые обновления на странице не вводятся — страница остаётся справочной (§11).
- Секция «Rule change parameters» сохраняется как есть (§7).
- Субсидия/стейкинг не «разбиваются по монетам» (§8).
- Ключ эмиссии SKA-монет в v1 не показывается (§9); более полная детализация прочих узловых полей сверх перечисленных — отложена.
- Пересчёт и атрибуция самих наград PoW/PoS не входят в область этой страницы — страница только отображает долю субсидии из конфигурации сети (§5).
- Кеширование по блокам и статичность данных страницы не меняются (§11).

## 14. Критерии приёмки

- [ ] Нигде на странице нет подписи `DCR`; суммы подписаны `VAR`.
- [ ] Ссылка «from chaincfg/params.go» ведёт на Monetarium, а не на Decred.
- [ ] Секция «Treasury parameters» полностью отсутствует; строка доли казны отсутствует в Subsidy.
- [ ] Доли награды PoW / PoS показывают фактические **50 % / 50 %** (не «60 % / 30 %»), согласованно с остальным обозревателем.
- [ ] Секция «Rule change parameters» присутствует и по смыслу не изменена.
- [ ] Таблица адресных префиксов показывает **корректные префиксы Monetarium** (mainnet → `M` / `Ms…`, не Decred `Dk / Ds…`); для ранее не поддержанных сетей таблица не пустая молча.
- [ ] Присутствует секция «SKA coin parameters» со всеми сконфигурированными SKA-монетами, по порядку номера типа; метка монеты в формате `SKA1` (без дефиса); активные/неактивные различимы.
- [ ] SKA-суммы (максимальная эмиссия, минимальная комиссия, масштаб монеты, опц. суммы эмиссии) показаны полностью и точно (18 знаков), без округления и без экспоненциальной записи; высота начала эмиссии — ссылка на блок, адреса эмиссии — ссылки на страницы адресов.
- [ ] Поведение «справочная статичная страница» сохранено: значения соответствуют конфигурации сети на момент запуска; страница ошибки при недоступности данных работает штатно.
- [ ] Тёмная тема корректно отрабатывает на изменённой странице (новая секция, изменённые таблицы).
