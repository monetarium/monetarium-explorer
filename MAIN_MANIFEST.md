# Спецификация Monetarium Explorer

Настоящий документ определяет требования к функциональности и интерфейсу обозревателя блоков Monetarium, построенного на базе форка `dcrdata`.

---

## 1. Технические основы и ограничения

### Поддержка токенов

- Система должна обеспечивать корректную работу и отображение до **255 типов монет SKA**.

### Разрядность и точность данных

- **Монеты VAR:**
  Используют точность 8 знаков в целой части и 8 знаков после запятой (аналогично DCR).

- **Монеты SKA:**
  Имеют повышенную разрядность — **15 знаков в целой части и 18 знаков после запятой**.

- **Особенность реализации:**
  Стандартные типы данных Go не поддерживают такую точность, поэтому для всех расчетов на бэкенде используется **специализированный модуль большой разрядности**.

---

## 2. Стандарты интерфейса (UI)

- **Правило трех цифр (Главная страница):**
  Для всех типов монет (VAR и SKA) на главной странице суммы отображаются с использованием **только трех значащих цифр** и буквенных сокращений (K, M, B, T).

- **Страницы деталей:**
  В деталях блоков и транзакций отображаются **все знаки после запятой** (8 для VAR и 18 для SKA).

- **Адаптивность:**
  Интерфейс строится по принципу **mobile-first** с обязательной поддержкой **темной темы**.

---

## 3. Главная страница

### 3.1. Таблица Latest Blocks

Таблица предназначена для отображения списка записей с агрегированными значениями и связанными транзакциями с возможностью раскрытия деталей на уровне строки.

---

### 3.1.2. Область отображаемых данных

#### 3.1.2.1. Основной уровень (строка)

Каждая строка должна содержать следующие поля:

---

##### Height

- **Сокращённый заголовок:** Height
- **Подсказка:** block height
- **Описание:** Порядковый номер блока

---

##### Transactions

- **Сокращённый заголовок:** Txn
- **Подсказка:** number of transactions
- **Описание:** Количество транзакций

---

##### VAR

- **Сокращённый заголовок:** VAR
- **Подсказка:** total VAR amount
- **Описание:** Общая сумма перечисленных монет VAR

---

##### SKA

- **Сокращённый заголовок:** SKA
- **Подсказка:** total SKA amount
- **Описание:** Общая сумма перечисленных монет SKA

---

##### Size

- **Сокращённый заголовок:** Size
- **Подсказка:** total block size
- **Описание:** Общий (суммарный) размер

---

##### Votes

- **Сокращённый заголовок:** Vote
- **Подсказка:** tickets voted
- **Описание:** Количество билетов, проголосовавших за включение данного блока в блокчейн

---

##### Tickets

- **Сокращённый заголовок:** Tkt
- **Подсказка:** tickets purchased
- **Описание:** Количество купленных билетов для голосования

---

##### Revokes

- **Сокращённый заголовок:** Rev
- **Подсказка:** tickets revoked
- **Описание:** Количество возвращённых (аннулированных) билетов, по причине неактивности кошелька данного билета во время голосования или истечения срока голосования для данного билета

---

##### Age

- **Сокращённый заголовок:** Age
- **Подсказка:** block age
- **Описание:** Время, прошедшее от момента включения этого блока в блокчейн до настоящего момента. Т.е. возраст блока.

---

#### 3.1.2.2. Уровень транзакций (expanded rows)

- Expanded rows используют **те же колонки**, что и основной уровень.
- Отдельные заголовки для них не предусмотрены.

---

##### 3.1.2.2.1. VAR строка

Значения должны интерпретироваться следующим образом:

- **Transactions (Txn):**
  Описание: Количество транзакций простого перечисления монет VAR

- **VAR:**
  Описание: Общая сумма перечисленных монет VAR

- **Size:**
  Описание: Суммарный размер транзакций VAR

---

##### 3.1.2.2.2. SKA-n строки

Для каждой строки SKA-n значения интерпретируются:

- **Transactions (Txn):**
  Описание: Количество транзакций простого перечисления монет SKA

- **SKA:**
  Описание: Общая сумма перечисленных монет SKA-n

- **Size:**
  Подсказка: SKA block size
  Описание: Суммарный размер транзакций SKA-n

---

### 3.1.3. Иерархия данных

Данные должны быть организованы в два уровня:

1. Основной уровень (строка)
2. Expanded rows:
   - одна строка VAR
   - множество строк SKA-n

Дополнительные уровни вложенности не допускаются.

---

### 3.1.4. Поведение раскрытия (expand/collapse)

#### 3.1.4.1. Раскрытие строки

- Пользователь должен иметь возможность раскрыть строку
- При раскрытии отображаются:
  - VAR
  - все SKA-n

#### 3.1.4.2. Состояние по умолчанию

- Строки должны быть свернуты

---

### 3.1.5. Подсчёт агрегированных значений

На основном уровне:

- Transactions = VAR + все SKA
- VAR = сумма VAR транзакций
- SKA = сумма всех SKA-n

---

### 3.1.6. Отсутствующие данные

Если значение отсутствует:

- должно отображаться явное обозначение отсутствия

---

### 3.1.7. Уникальность и идентификация

- Каждая строка имеет уникальный Height
- Каждая запись связана с одной строкой
- Каждый SKA-n идентифицируется уникальным номером n

---

### 3.1.8. Сортировка

- Основные строки сортируются по Height
- SKA-n строки сортируются по номеру n

---

### 3.1.9. Масштабируемость

Система должна поддерживать:

- большое количество строк
- до 255 SKA-n на строку

---

### 3.1.10. Навигация

- Height должен поддерживать переход к деталям блока

---

### 3.1.11. Ограничения

- Отсутствует агрегированная строка SKA в expanded rows
- Нет вложенности глубже одного уровня
- Все SKA-n отображаются при раскрытии

---

### 3.1.12. Расширяемость

Система должна позволять:

- добавление новых типов транзакций
- расширение набора метрик

---

#### 4. Раздел Mempool

Раздел предназначен для отображения транзакций, ожидающих включения в блок, с визуализацией прогнозируемого распределения места.

##### 4.1. Изменения в таблице транзакций

- **Типы транзакций:** Тип `Regular` заменяется на `VAR`. Транзакции перечисления токенов SKA обозначаются как `SKA-1`, `SKA-2` и т.д..
- **Заголовки колонок:** Колонка `DCR` переименовывается в `Coins` или `Amount`.
- **Удаляемые элементы:** Из интерфейса удаляются старые элементы анимации и хеши, заменяясь на новый индикатор заполненности.

##### 4.2. Индикатор распределения блока

Вместо стандартной анимации вводится горизонтальный индикатор (полоса), отображающий ожидаемое использование пространства в следующем блоке.

- **Параметры полосы:** Общая длина соответствует максимально допустимому размеру блока (393,216 байт).
- **Гарантированное пространство:**
  - **VAR:** 10% блока.
  - **SKA:** 90% блока, разделяемые поровну между всеми эмитированными типами монет SKA.
- **Цветовая индикация сегментов:**
  - **Серый:** Свободное место.
  - **Зелёный:** Транзакции монеты укладываются в гарантированный лимит.
  - **Жёлтый:** Превышение лимита, но транзакции будут включены за счет свободного места других монет.
  - **Красный:** Превышение лимита, при котором не все транзакции этой монеты попадут в следующий блок.

---

#### 5. Дополнительные блоки главной страницы

##### 5.1. Раздел Voting (Голосование)

- **Вознаграждения VAR:** Подраздел `Vote Reward` переименовывается в `Vote VAR Reward`. Отображает доходность на одну застейканную монету VAR (за последний блок, 30 дней и год).
- **Вознаграждения SKA:** Добавляется подраздел `Vote SKA Reward`. Отображает доходность в монетах SKA-n на одну застейканную монету VAR. Создается отдельный блок для каждого эмитированного типа SKA.

##### 5.2. Раздел Mining (Майнинг)

- **PoW VAR Reward:** Заменяет `POW Reward`. Заголовок индикатора изменения сложности меняется на `Next Block VAR Reward Reduction`.
- **PoW SKA Reward:** Новый подраздел, отображающий сумму вознаграждения в монетах SKA за создание последнего блока. По одной строке для каждого типа SKA.

##### 5.3. Раздел Supply (Эмиссия)

Заменяет собой раздел `Distribution`.

- **VAR Coin Supply:** Отображает текущее количество монет VAR в обращении относительно целевого показателя (~47M).
- **SKA Coin Supply:** Для каждого типа SKA отображается:
  - **In Circulation:** Текущее количество в обращении.
  - **Total Issued:** Всего эмитировано.
  - **Total Burned:** Всего выведено из обращения (сожжено/отозвано).
