## Требования к функции: Таблица страницы Blocks — поддержка монет SKA

### Контекст

Страница `/blocks` с пагинацией использует унаследованный от Decred макет таблицы, в котором отображается единственная колонка со значением `DCR` без учёта типов монет. Таблица Latest Blocks на главной странице уже обновлена для поддержки двухмонетной модели (строки VAR + SKA-n, раскрывающийся аккордеон). Данная функция приводит таблицу страницы Blocks в соответствие с этой моделью, сохраняя при этом колонки, уникальные для страницы Blocks: `Version` и `Time (UTC)`.

---

### 1. Структура колонок

Обновлённая таблица Blocks должна содержать следующие колонки в указанном порядке:

| Колонка      | Подпись (десктоп) | Подпись (мобильный/сокращённая) |
| ------------ | ----------------- | ------------------------------- |
| Height       | Height            | Height                          |
| Transactions | Transactions      | Txn                             |
| Votes        | Votes             | -                               |
| Tickets      | Tickets           | -                               |
| Revocations  | Revocations       | -                               |
| VAR          | VAR               | VAR                             |
| SKA          | SKA               | SKA                             |
| Size         | Size              | Size                            |
| Version      | Version           | -                               |
| Age          | Age               | Age                             |
| Time (UTC)   | Time (UTC)        | Time (UTC)                      |

Колонка `DCR` заменяется колонками `VAR` и `SKA`.

---

### 2. Поведение раскрывающихся строк

Каждая строка блока должна быть раскрываемой — идентично поведению таблицы Latest Blocks:

- **Состояние по умолчанию:** свёрнуто.
- **По клику:** строка раскрывается и показывает одну под-строку VAR и одну под-строку SKA-n на каждый тип SKA, присутствующий в блоке.
- Под-строки используют те же колонки, что и родительская строка. Отдельные заголовки для под-строк не предусмотрены.
- В под-строках не отображаются значения в колонках Votes, Tickets, Revocations, Version, Age и Time — в этих ячейках показывается `—`.

Данные под-строк по типу монеты:

**Под-строка VAR:**

- Transactions: количество обычных (Regular) VAR-транзакций в блоке
- VAR: общее количество монет VAR в блоке
- Size: размер VAR-транзакций в байтах

**Под-строка SKA-n (по одной на каждый присутствующий тип):**

- Transactions: количество обычных (Regular) SKA-n-транзакций в блоке
- SKA: общее количество монет SKA-n в блоке
- Size: размер SKA-n-транзакций в байтах

---

### 3. Колонка Transactions — родительская строка

Количество транзакций в колонке `Transactions` родительской строки должно исключать типы Vote, Ticket и Revocation — в соответствии со спецификацией `MAIN_MANIFEST.md §3.1.1.2` и поведением таблицы Latest Blocks. Голоса, билеты и возвраты уже вынесены в отдельные колонки.

---

### 4. Значения VAR и SKA — точность отображения

Суммы отображаются по правилу трёх цифр (`wiki/foundation/ui-standards.md §1`): три значащие цифры с буквенными сокращениями (K/M/B/T). Пример: `4.67k`, `2.94k`.

Если в блоке нет SKA-транзакций, в колонке SKA отображается `—`.

---

### 5. Шеврон / элемент раскрытия

На каждой строке должен присутствовать индикатор-шеврон (аналогично Latest Blocks), сигнализирующий о возможности раскрытия. При раскрытии/свёртывании направление шеврона меняется.

---

### 6. Колонки Version и Time (UTC)

Эти колонки уникальны для страницы Blocks и должны быть сохранены:

- `Version` отображается только в родительской строке. В под-строках — `—`.
- `Time (UTC)` отображается только в родительской строке. В под-строках — `—`.
- `Age` — относительное время, рендерится через JS (тот же механизм `data-time-target="age"`, что и в Latest Blocks и существующей таблице Blocks).

---

### 7. Адаптивное поведение

Следовать той же логике контрольных точек (breakpoints), что и в таблице Latest Blocks и существующей таблице Blocks.

---

### 8. Вне области действия

- Логика пагинации не изменяется.
- Порядок сортировки не изменяется (блоки остаются отсортированы по Height по убыванию).
- Ссылки `/side` (orphaned blocks) и `/disapproved` (PoS invalidated blocks) внизу страницы остаются без изменений.

---

### 9. Критерии приёмки

- [ ] Каждая строка блока раскрывается и показывает под-строки VAR + SKA-n по клику.
- [ ] Колонка `DCR` заменена колонками `VAR` и `SKA`.
- [ ] Количество транзакций в родительской строке исключает типы Vote, Ticket и Revocation.
- [ ] Блоки без SKA-транзакций показывают `—` в колонке SKA.
- [ ] Блоки с SKA-транзакциями показывают корректные под-строки SKA-n при раскрытии.
- [ ] Колонки `Version` и `Time (UTC)` присутствуют и не затронуты механизмом раскрытия.
- [ ] Брейкпоинты адаптивности и тёмная тема сохранены.
- [ ] Отображение сумм следует правилу трёх цифр (суффиксы K/M/B/T).
